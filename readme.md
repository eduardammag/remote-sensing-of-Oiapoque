# Passo a Passo do Processamento das Layers

## 1. Carregar a grade
- O arquivo da grade (`grade_path`) é lido com `geopandas`.
- A grade define as células (polígonos) que serão usadas para analisar as layers.

## 2. Reprojetar a grade (CRS)
- Se `crs_target` for fornecido, a grade é reprojetada para esse sistema de coordenadas.
- Garante que as medidas de área e distância sejam consistentes (em metros).

## 3. Verificação de unidades
- A função checa se o CRS está em unidades métricas.
- Isso garante que as métricas espaciais façam sentido.

## 4. Construir mapa de classes
- A função `build_class_code_map(layer_list)` cria um mapeamento de códigos para cada classe em cada layer.
- Facilita o cálculo de métricas padronizadas para todas as layers.

## 5. Carregar cada layer
Para cada tupla `(caminho_arquivo, nome_coluna_classe)` em `layer_list`:
- Ler a camada com `geopandas`.
- Reprojetar para o CRS da grade, se necessário.
- Armazenar em `layer_gdfs` junto com o nome da coluna de classe.

## 6. Processar layers auxiliares (estradas/mineração)
Se forem fornecidas:
- Ler a camada.
- Reprojetar para CRS da grade.
- Unir todas as geometrias em uma única (`unary_union`) para facilitar o cálculo de distância.

## 7. Preparar tarefas de processamento
- Cada célula da grade vira uma “tarefa”: `(índice da célula, geometria da célula)`.
- Limitar número de células se `test_n` for definido (modo teste).

## 8. Processar cada célula
Função `process_cell` recebe:
- Geometria da célula
- Todas as layers (`layer_gdfs`)
- Mapa de classes
- Resolução espacial

Calcula métricas de fragmentação para cada classe dentro da célula:
- Área total por classe
- Número de fragmentos
- Forma, bordas, densidade, etc.

## 9. Processamento paralelo (opcional)
- Se `n_workers > 1`, as células são processadas em paralelo usando `ProcessPoolExecutor`.
- Caso contrário, processa uma célula de cada vez.

## 10. Consolidar resultados
- Combina os resultados de todas as células em um `DataFrame`.
- Salva métricas por célula em CSV.
- Junta métricas com a geometria da grade para gerar GeoPackage (GPKG).

## 11. Calcular distâncias a feições auxiliares
Para cada célula, calcula a distância do centróide até:
- Estradas (se `road_union` existir)
- Áreas de mineração (se `mining_union` existir)

- Adiciona essas informações como colunas no `GeoDataFrame`.

## 12. Salvar resultados finais
- CSV com métricas por célula.
- GeoPackage com métricas + geometria + distâncias.
- Retorna dicionário com caminhos e DataFrames.

# CRM: 
significa Coordinate Reference System — ou sistema de referência de coordenadas. 
Um CRS métrico é aquele em que as coordenadas estão expressas em metros, e não em graus de latitude/longitude.
Isso importa, pois qualquer cálculo de distância, área, buffer, ou tamanho de grade (como sua grade de 2000 m) só faz sentido se os valores estiverem em metros.


# UTM (Universal Transverse Mercator):
é um sistema de coordenadas projetado:
- divide o mundo em 60 zonas
- cada zona tem 6° de longitude
- usa metros como unidade
- distorce pouco áreas pequenas e médias

### Exemplo de UTM para o Amapá (onde fica Oiapoque):
Oiapoque está próximo da zona UTM 22N / 22S
Código EPSG típico para o Amapá:
- EPSG:31972 → SIRGAS 2000 / UTM zone 22N
- EPSG:31978 → SIRGAS 2000 / UTM zone 22S
o uso de UTM aqui é ideal, pois queremos uma grade de 2000 m × 2000 m.
Se o dado estivesse em lat/long (graus), 2000 significaria 2000 graus, o que é absurdo.


# Rasterizar:
significa converter dados vetoriais em uma grade de células (raster).
- Dados vetoriais: representam objetos como pontos, linhas e polígonos, com coordenadas precisas (ex.: um polígono que representa um lago).
- Dados raster: representam a superfície como uma matriz de células (pixels), cada uma com um valor (ex.: 0 para terra, 1 para água).
Em outras palavras: você pega um polígono ou linha e “pinta” ele em uma grade regular de pixels.

Exemplo: Imagine que você tem um polígono representando uma floresta.
- Vetorial: você tem as coordenadas exatas do polígono.
- Raster: você cria uma grade, e cada célula que toca a floresta recebe o valor 1 (floresta), enquanto as outras células recebem 0.

# Patch: 
é uma porção contínua do mesmo tipo de cobertura, sem interrupções.
Ele pode ser representado em vetorial como um polígono ou em raster como conjunto de pixels conectados com o mesmo valor.
Exemplos: 
- Floresta: Uma área contínua de floresta em um mapa é um patch. Se houver outra floresta separada por um rio ou estrada, é outro patch.
- Água: Um lago grande é um patch. Um pequeno lago distante é outro patch, mesmo que sejam da mesma classe “água”.

As métricas de paisagem (como as que você está calculando: número de patches, área média, maior patch, shape index) dependem dos patches.
Eles ajudam a medir: fragmentação da paisagem, continuidade da vegetação, complexidade espacial, 
                     Raster vs Vetorial (
                            Raster: patches = conjuntos de pixels conectados (4 ou 8 vizinhos). 
                            Vetorial: patches = polígonos contínuos com a mesma classe.)
